## About my approach

I'm a Business Analyst, not a data scientist — my contribution on data and ML projects is the **translation layer**: defining the business problem, specifying analytical requirements, setting model acceptance criteria, facilitating decisions on outputs, and turning results into stakeholder-ready recommendations. The code here demonstrates that I can work hands-on with the data to support those decisions, not that I operate as a full-time ML engineer.
